The environment contains a long technical report, bundled archive artifacts, and a Java analysis scaffold for a scientific-computing benchmark reproduction task. This is a computational-imaging and medical-informatics benchmark for melanoma triage, using archived predictions from a vision-model evaluation to reconstruct the accepted numerical summary of a published experiment. The task centers on numerical evaluation, threshold analysis, reproducibility, and medical-image classification rather than a general implementation exercise. The analysis program must produce a JSON object at /app/out/reproducibility.json that records the accepted benchmark summary for the provided fixture. The output must follow the schema in /app/docs/repro-schema.json and include the model variant, threshold, warmup policy, Analysis profile, F1 score, latency, and source.

The required values are encoded across the report text and the archived prediction files. The benchmark concerns an image-classification workflow for melanoma triage, a model-comparison experiment, and a reproducibility target that must be reconstructed from archived evidence rather than from a fresh simulation. The solution should satisfy the contract exactly for the supplied fixture, with deterministic output and no dependency on re-running the original computation.
